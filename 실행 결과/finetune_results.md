# RawNet2 한국어 파인튜닝 진행 상황

## 데이터셋 구성

### 영어 (사전학습)
- **ASVspoof 2019 LA** — 영어 화자 진짜/가짜 음성

### 한국어 (파인튜닝)
| 구분 | 데이터셋 | 설명 |
|---|---|---|
| 진짜 (label=0) | 자유대화 음성(일반남녀) — 스튜디오, 스튜디오_1, 스튜디오_2 | 실제 사람 녹음 |
| 가짜 (label=1) | 감성 및 발화 스타일별 음성합성 데이터 | 성우 녹음 문장을 TTS로 변환 |

스튜디오 폴더만 사용하는 이유: 다른 환경(AI챗봇, 음성수집도구)은 잡음이 많아 가짜 데이터와 품질 차이가 크기 때문.
모델이 "잡음 있으면 진짜, 없으면 가짜"를 학습하는 것을 방지하기 위해 최대한 깨끗한 음성만 사용.

---

## 데이터 전처리

### 문제
- 가짜(TTS) 데이터는 스튜디오급으로 소음이 전혀 없음
- 진짜(스튜디오) 데이터도 깨끗하지만 일정한 배경 소음(에어컨 등)이 존재
- 이 차이를 모델이 학습하면 소음 유무로 판별하는 잘못된 패턴이 생길 수 있음

### 해결
- `preprocess_denoise.py` 로 진짜 음성 전처리 (1회 실행)
- `noisereduce` 라이브러리의 `stationary=True` 모드로 일정한 배경 소음 제거
- 처리된 파일을 `Training_denoised` 폴더에 저장 (원본 유지)
- `.wav`와 함께 `.npy` 형식으로도 저장 → 학습 중 로드 속도 약 10배 향상
- 학습 시 `.npy`가 있으면 자동으로 우선 사용, 없으면 `.wav` fallback

## 파인튜닝 전략

### 영어 데이터 혼합 (Catastrophic Forgetting 방지)
- 한국어만 학습하면 기존 영어 탐지 능력이 소실됨
- 영어:한국어 = 2:8 비율로 혼합하여 학습

### 단계적 레이어 해제 (Stage)
기존 가중치 손상을 막기 위해 3단계로 나눠서 학습.

| Stage | 학습 레이어 | 학습률 | 권장 에폭 |
|---|---|---|---|
| 1 | GRU + FC만 | 1e-4 | 10~15 |
| 2 | 중간 레이어 추가 해제 | 1e-5 | 10~15 |
| 3 | 전체 레이어 | 1e-6 | 5~10 |

### Early Stopping
- `PATIENCE=5` : 5에폭 연속으로 EER 개선이 없으면 자동 종료
- 불필요한 학습 시간 단축 및 오버피팅 방지

## Validation 폴더 미사용 이유
466번 데이터셋의 Validation 폴더는 사용하지 않음.
Training 데이터(가짜 652,393개, 진짜 46,568개)가 충분하여 코드 내에서 8:2로 분할해 검증 세트를 구성함.
별도 Validation 폴더를 사용하면 학습 데이터가 늘어나고 검증 신뢰도가 높아지는 장점이 있으나,
현재 데이터 규모에서는 8:2 분할로도 충분한 검증이 가능하다고 판단.
