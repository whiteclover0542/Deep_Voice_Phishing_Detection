## 전처리

진짜 음성에 배경 소음(에어컨 등)이 있어 가짜(무잡음 TTS)와 품질 차이 발생 → 모델이 잡음 유무로 판별할 위험.
`preprocess_denoise.py`로 1회 전처리 (`noisereduce stationary=True`). `.wav` + `.npy` 동시 저장 (로드 속도 10배).

## 파인튜닝 전략

### Stage (단계적 레이어 해제)

위(분류기)→아래(특징 추출기) 방향으로 점진적으로 해제. 한 번에 전체를 높은 학습률로 학습하면 사전학습 지식이 파괴됨(Catastrophic Forgetting).

| Stage | 동결 | 학습 | 학습률 | PATIENCE |
|---|---|---|---|---|
| 1 | sinc, bn0, blocks 전체 | GRU + FC | 1e-4 | 3 |
| 2 | sinc, bn0, blocks.0~3 | blocks.4~5 + GRU + FC | 1e-5 | 4 |
| 3 | 없음 | 전체 | 1e-6 | 5 |

> Stage가 올라가도 466번 단일 TTS 데이터면 EER은 낮게 유지됨 — 정상. 실제 일반화 성능은 다른 TTS 엔진으로 검증해야 함.

### 기타
- **영어 혼합**: 영어 20% + 한국어 80% (Catastrophic Forgetting 방지)
- **tel_aug**: 진짜/가짜 양쪽에 50% 확률로 전화망 효과 적용 (대역통과 + 코덱 + 패킷 손실 + AGC)

## EER 0% 문제

**원인**: 466번 TTS는 단일 엔진 → 모든 샘플이 동일한 보코더 지문 공유. RawNet2가 사전학습에서 이미 학습한 패턴이라 파인튜닝 없이도 완벽 분류.

**해결**: tel_aug 적용 → 전화 채널 효과가 보코더 지문을 덮어 두 분포가 겹침 → EER 현실화.

| | tel_aug 전 | tel_aug 후 |
|---|---|---|
| real score 평균 | 0.0008 (std=0.003) | 0.4285 (std=0.035) |
| fake score 평균 | 0.9985 (std=0.004) | 0.6045 (std=0.163) |
| EER | 0.00% | 10.14% |

**함께 수정된 버그**

| 버그 | 원인 | 수정 |
|---|---|---|
| 데이터 누수 | train/dev 호출마다 독립 shuffle → 동일 파일이 양쪽 포함 | `random_split`으로 고정 분할 |
| Freeze 미작동 | FREEZE_MAP 키워드가 실제 파라미터명과 불일치 | 키워드 수정 (`layer1` → `blocks.0` 등) |
