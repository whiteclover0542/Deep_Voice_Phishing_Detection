# STFT 스펙토그램 / MFCC 히트맵 분석

## 1. 두 결과 비교

### 가짜 (AI 목소리) — 가짜 100%
- **STFT**: 고음역(3kHz 이상)이 전 구간에 걸쳐 균일하게 빽빽하게 채워져 있음. 수직 줄무늬가 일정한 간격으로 반복됨
- **MFCC**: 색이 전체적으로 단조롭고 변화가 적음. 아래 두 행은 극단적인 값(진한 빨강/파랑)이 계속 유지됨

### 진짜 (사람 목소리) — 진짜 88.2%
- **STFT**: 고음역이 띄엄띄엄 불규칙하게 분포. 검은 구간(침묵/호흡)이 자연스럽게 섞여 있음
- **MFCC**: 시간에 따라 색상이 다양하고 들쭉날쭉하게 변함

---

## 2. STFT 스펙토그램이란

음성을 아주 짧은 구간으로 잘라서, 각 구간마다 "어떤 주파수가 얼마나 울리고 있나"를 측정한 것.

- **X축** = 시간, **Y축** = 주파수(Hz), **밝기** = 에너지 세기
- 사람은 숨을 쉬고 강세가 오르내리기 때문에 자연스러운 불규칙성이 생김
- AI는 알고리즘이 끊김 없이 생성하기 때문에 고음역이 지나치게 고르게 채워짐

---

## 3. MFCC 히트맵이란

사람의 청각 특성을 반영해 음성의 특징을 13개 숫자로 압축한 것.

- **X축** = 시간, **Y축** = 계수 번호 (0번~12번)
- 각 숫자는 "이 시점에 음성이 어떤 소리 모양인가"를 나타냄
- 사람 목소리는 감정·호흡·강세에 따라 값이 불규칙하게 변함
- AI 음성은 TTS가 매끄럽게 생성하므로 패턴이 단조롭거나 반복적으로 보임

---

## 4. 멜 스펙토그램을 왜 쓰지 않는가

멜 스펙토그램은 **사람이 설계한 고정 공식**이다.

> 멜 스케일(Mel scale)은 사람의 귀가 주파수를 인식하는 방식을 수식으로 정리한 것으로,  
> 어떤 음성에 적용해도 항상 **동일한 방식으로 변환**된다.  
> 학습으로 조정되는 것이 아니라, 인간이 미리 정해둔 처리 방식이다.

### 문제점
이 변환 과정에서 **정보가 뭉개진다.**  
딥페이크 탐지의 핵심은 AI 음성에 남은 아주 미세한 인공적 흔적인데,  
멜 스펙토그램으로 변환하면 그 흔적이 사라질 수 있다.

### 이 프로젝트의 방식 (RawNet2)
원본 파형 그대로 입력받아 **SincConv가 직접 어떤 주파수를 볼지를 학습**한다.

| | 멜 스펙토그램 | RawNet2 (SincConv) |
|---|---|---|
| 필터 설계 | 사람이 고정 설계 | 모델이 학습으로 결정 |
| 정보 손실 | 변환 과정에서 발생 | 원본 파형 그대로 유지 |
| 딥페이크 탐지 | 미세한 흔적 놓칠 수 있음 | 미세한 패턴까지 포착 가능 |
